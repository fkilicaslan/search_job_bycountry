# Evaluation Scoring Rubric

Use when scoring job evaluations to ensure consistent, honest assessments.

---

## Score dimensions (base weights)

| Dimension | Weight | What it measures |
|---|---|---|
| Hard requirement coverage | 30% | % of required qualifications matched by real experience |
| Seniority alignment | 15% | Candidate level vs. role level (over/under-qualified) |
| Domain / industry match | 10% | Relevant industry experience |
| Compensation alignment | 10% | Candidate target vs. stated/estimated range |
| Location / remote fit | 10% | Geographic and work arrangement match |
| Credentials / language | 10% | Required certifications, language proficiency, clearances |
| Skills / tools | 10% | Specific skills and platforms required |
| Career trajectory fit | 5% | Does this role make sense as the next logical step? |

---

## Role-type weight adjustments

| Role type | Adjustments |
|---|---|
| **Sales / Enterprise AE** | Hard requirements reframed as "track record" — quota attainment and new logo evidence replace formal skill checks; Credentials -5%, Skills -5%, Hard req +10% |
| **Sales Director / VP** | Seniority +10%, Domain +5%, Skills -15% — scope, P&L, and team leadership matter most |
| **CSM / Customer Success** | Domain +5%, Skills +5%, Seniority -5%, Trajectory -5% |
| **BD / Partnerships** | Hard req +5%, Domain +5%, Credentials -5%, Trajectory -5% |
| **Pre-Sales / Solutions Engineer** | Skills +10%, Hard req +5%, Credentials -5%, Trajectory -10% |
| **Executive / C-Suite** | Seniority +10%, Domain +5%, Skills -15% |

---

## Instant SKIP / hard-cap rules

These override the weighted score regardless of other dimensions:

| Condition | Action |
|---|---|
| Required language not in candidate profile (e.g. French native, Spanish required) | Cap score at **2.0** — flag prominently |
| US-only role, no remote, candidate Berlin-based with no visa | Cap score at **2.5** |
| Required credential completely absent (bar admission, clearance, medical license) | Cap score at **2.0** |
| Seniority mismatch ≥ 2 levels (e.g. 20yr director vs. 2yr IC requirement) | Flag overqualification risk; reduce Seniority dimension by 50% |

## Score thresholds and recommended actions

| Score | Label | Recommended action |
|---|---|---|
| 4.5–5.0 | Excellent Match | Generate CV + cover letter immediately |
| 3.5–4.4 | Good Match | Generate CV + cover letter |
| 3.0–3.4 | Worth Considering | Full evaluation, flag risks clearly, user decides |
| 2.0–2.9 | Weak Match | Warn user, suggest better-matched roles |
| 1.0–1.9 | Poor Match | Advise against — explain why, suggest alternatives |

For scores below 3.0, be direct:
> "This is a stretch. The main gap is {X}. Your time is better spent on roles that match your {strength}."

---

## Scoring rules

1. **Never inflate scores.** A 3.2 is a real score. Honesty serves the user better than cheerleading.
2. Hard-cap rules override all dimensions in regulated or blocked situations.
3. If profile lacks data to score a dimension, score it neutral (5/10) and flag "incomplete data."
4. Compensation misalignment (target 30%+ above range) does not kill the score but MUST be flagged.
5. "Years of experience" requirements are guidelines, not hard cutoffs. 5 years against a 7-year requirement is a partial match, not a disqualification.
6. Career break (2024–2025) is a context item, not a negative score modifier — the AI/Scrum/language upskilling period is relevant and should be framed as intentional.
